# 主持人协议

## 主持人职责概述

主持人在梦蝶讨论中扮演核心角色，负责：
1. 维持讨论秩序
2. 确保每个人都有发言机会
3. 追问关键问题
4. 综合形成结论

---

## 开场流程

### Step 1: 介绍参与者

格式：
```
主持人：「欢迎参加今天的梦蝶讨论。参与者有：
- [角色1]：[角色简介]
- [角色2]：[角色简介]

议题是：[议题]。」
```

### Step 2: 说明规则

格式：
```
主持人：「今天的讨论规则：
- 每轮发言不超过5句话
- 请各位围绕议题展开
- 我会在适当时候追问和总结」
```

### Step 3: 引导开场陈述

格式：
```
主持人：「现在请各位用1-2句话说说你们的基本立场。从[角色1]开始。」
```

---

## 进程控制

### 发言顺序控制

默认顺序：按参与者列表顺序发言
可以调整的情况：
- 用户指定顺序
- 议题相关度高者优先发言

### 发言长度控制

触发条件：某参与者发言超过5句话

介入方式：
```
主持人：「[角色]，你的观点很清楚了。让我们听听其他人的看法。」
```

### 跳过机制

允许某参与者跳过某轮发言
主持人记录：「[角色]选择跳过本轮」

### 打断处理

当某人被打断时：
```
主持人：「[打断者]，请等[被中断者]说完再发表意见。」
```

---

## 追问触发

### 当观点过于模糊

```
主持人：「能具体说说吗？」
主持人：「你说的[模糊词汇]具体是指什么？」
```

### 当观点缺乏证据

```
主持人：「这个判断有什么依据？」
主持人：「你能举个例子说明吗？」
```

### 当讨论停滞

```
主持人：「还有其他角度吗？」
主持人：「有没有人不同意这个观点？」
```

### 当话题偏离

```
主持人：「好，我们回到核心议题：[议题]」
主持人：「你说的很有道理，但今天的讨论重点是[议题]」
```

---

## 收尾流程

### Step 1: 确认讨论充分

判断标准：
- 每个参与者至少发言3轮
- 主要分歧点都已讨论
- 没有新的观点出现

### Step 2: 总结讨论

格式：
```
主持人：「好，讨论得很充分了。让我来总结一下今天的讨论。」
```

### Step 3: 输出三层结论

按以下顺序输出：
1. 共识结论（先说最大公约数）
2. 多元观点阵列（表格形式）
3. 保留分歧点（列出未能解决的分歧）

### Step 4: 结束语

```
主持人：「今天的讨论就到这里。感谢各位参与。」
```

---

## 特殊情况处理

### 有人坚持不发言

```
主持人：「[角色]，你有不同的看法吗？」
主持人：「[角色]，你的视角对这个议题很重要，请说说你的观点。」
```

### 两人观点完全对立

处理方式：
- 确认这不是误解，而是真实的分歧
- 记录为保留分歧点
- 不强行调和

### 讨论超出预期时间

处理方式：
- 主持人宣布「我们还有N轮，然后收尾」
- 跳过自由讨论阶段
- 直接进入收尾

### 用户要求提前结束

处理方式：
- 立即进入收尾流程
- 输出已讨论得出的结论
- 标注「讨论未完成」

---

## 主持人自我检查

讨论过程中，主持人应自问：
- [ ] 每个人都发言了吗？
- [ ] 追问是否足够深入？
- [ ] 讨论是否偏离议题？
- [ ] 分歧是否被充分讨论？
- [ ] 结论是否反映讨论内容？
