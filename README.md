## 核心定位

梦蝶取自庄周梦蝶的典故，象征多视角的交融与思辨。

梦蝶的核心使命是将多个视角 Skill 组织成讨论模式，对指定议题进行讨论，产出有价值的结论。

## 与辩论的本质区别

| 辩论 | 梦蝶 |
|------|------|
| 分胜负，一方赢一方输 | 求理解，保留分歧 |
| 消灭异见 | 尊重差异 |

## 三层结论结构

1. **共识结论**：最大公约数，1-2句话
2. **多元观点阵列**：表格形式，保留重要分歧
3. **保留分歧点**：列出未能解决的重要分歧（含各方立场摘要）

## 触发词

启动梦蝶：
- 「开一个梦蝶讨论」
- 「开个会吧」
- 「让XX和XX讨论一下」
- 「梦蝶，议题是XX」
- 「来讨论一下XX」

扩展触发词：
- 「继续上次讨论」→ 读取最近的讨论记录

## 执行流程（详细见 SKILL.md）

| Phase | 内容 | 关键动作 |
|-------|------|---------|
| 0 | 启动判断 | 判断完整启动/引导补充/续接讨论 |
| 1 | 确认参数 | 议题、参与者、主持人、**Skill 验证** |
| 2 | 讨论执行 | 开场→第一轮→第二轮→自由讨论→收尾 |
| 3 | 结论输出 | **先保存**→输出结论 |
| 4 | 安可 | 用户追问1-3轮 |
| 5 | 质量自检 | 内部检查，不输出 |

## 质量标准

| 检查项 | 通过标准 |
|--------|---------|
| 讨论轮次 | 每位参与者至少发言3轮 |
| 观点深度 | 每轮至少3句话有实质内容 |
| 主持人追问 | 至少追问2次 |
| 共识结论 | 真实反映讨论，不是和稀泥 |
| 保留分歧 | 真实存在，非刻意制造 |

## 讨论实录

梦蝶产出的不是辩论记录，而是多视角交融的思考。以下摘录两场讨论的精华片段：

### 爱情讨论：命中注定 vs 日久生情

> **七龙珠孙悟空**：（摇摇头）不对。爱情不是想出来的，是感受出来的。就像布尔玛当年和我一起冒险，后来克林和我一起训练，这些都是真实的。不是想，是真的在一起经历过。

> **紫霞仙子**：对啊，经历。但在那之前，先要认定那个人。不是等到经历了才确定，是确定了就不动摇。

*——紫霞与七龙珠悟空的核心分歧：认定是瞬间的还是渐进的？*

### 产品讨论：Real artists ship

> **乔布斯**：产品是给用户的礼物。你做一个东西，然后把它交到用户手上，这是世界上最神圣的事。不好的产品，是对用户的侮辱。好的产品，是对用户的尊重。

> **马斯克**：Trump说的是营销不是产品，火箭成功了或者没有，不靠声誉。真正的产品烂了公司就死，Tesla 2008年差点死了三次因为产品好才活下来。

> **野兽先生**：好产品= ship出去+对用户产生了正向影响。两个都要。

*——四位参与者对"好产品"达成唯一共识：必须到达用户手中。*
